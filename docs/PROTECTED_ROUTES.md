# Защищенные маршруты

## Обзор

Все приватные страницы приложения теперь защищены от доступа неавторизованных пользователей с помощью компонента `ProtectedRoute`.

## Защищенные страницы

- `/dashboard` - дашборд с проектами пользователя
- `/create` - мастер создания нового проекта
- `/projects/[id]` - детальная страница проекта

## Реализация

### Компонент ProtectedRoute

Расположение: `frontend/components/auth/ProtectedRoute.tsx`

**Функциональность:**
1. Проверяет статус авторизации пользователя через `useAuth()` хук
2. Показывает индикатор загрузки во время проверки
3. Перенаправляет неавторизованных пользователей на главную страницу (`/`)
4. Отображает дочерние компоненты только для авторизованных пользователей

### Использование

```tsx
import { ProtectedRoute } from "@/components/auth/ProtectedRoute";

export default function SomePage() {
    return (
        <ProtectedRoute>
            {/* Контент доступен только авторизованным пользователям */}
        </ProtectedRoute>
    );
}
```

## Безопасность

- **Frontend**: Компонент `ProtectedRoute` предотвращает отображение приватных страниц
- **Backend**: Все API эндпоинты защищены через JWT guards на уровне контроллеров

Двухуровневая защита обеспечивает:
1. Хороший UX (пользователь сразу перенаправляется)
2. Реальную безопасность (данные не отдаются без валидного токена)
